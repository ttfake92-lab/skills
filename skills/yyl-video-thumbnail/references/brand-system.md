# Brand System —— YouTube 高设计感视频封面美学

这是整个 skill 的护城河。封面好不好看、是不是"一眼是这个频道"，全看这套规则有没有被守住。**每次开工前读一遍。**

## 0. 北极星

> 不是把信息塞进图里，是用一张图让人 0.3 秒内想点开。
> 缩略图在手机上只有指甲盖大小，**远看能立住**才算成功——眯眼只剩：一个人 + 一句大字 + 一个黄块。

## 1. pop-out 技法（本频道的核心方法，不能破）

三层，从下到上：

```
① 底层 = 人物的原始照片
   压暗 + 降饱和（brightness ≈ .66, saturate ≈ .95）
   保留房间 / 桌面 / 显示器等环境元素 → 真实感
   ⚠️ 适度压暗，别压死。环境要看得见。

② 中层 = 大标题
   压在暗化原图上；关键词 = 黄色实色块

③ 顶层 = 抠图人物
   全亮、原位精确叠回去（和底层原图用同一 object-position）
   人物会盖住一点文字 —— 这正是"从场景里跳出来"的纵深感来源
```

**铁律**：`photo`(原图) 和 `person_image`(抠图) 必须是同一张；抠图要**整幅输出**(不裁外框)才能对齐。

## 2. 视觉层级

1. **人物**（全亮跳出）—— 第一主体、真人感、频道辨识度。
2. **标题** —— 第二主体，关键词黄块。
3. **环境 / kicker / 品牌签名** —— 第三层，衬托，不抢。

人物可以盖住部分文字，**但关键词必须仍可读**（被挡到只剩半个词就要缩字号 / 调 focus）。

## 3. 配色

| 角色 | 值 | 用法 |
|---|---|---|
| 标题主色 | `#FFFFFF` | 大字 |
| **招牌色块（popout）** | `#FFE14D`(黄) | 关键词实色块 = 频道记忆点 |
| **强调蓝（ui / 点缀）** | `#3B82F6` / 选中 `#2563EB` | ui 模板关键词、命令条图标/光标、选中结果条 |

铁律：白大字 + 黄实色块(popout) / 蓝(ui) 是基本盘，保持稳定才有品牌感。一张图主色 ≤ 3 个。
**强调色只用蓝，不用青(#22D3EE 已弃)；少发光、克制。禁止绿色大字。**
> 改调性 = 改 `assets/template-popout.html` / `assets/template-ui.html` 顶部 `:root` 令牌。

## 4. 字体（设计感的关键）

- **拉丁关键词**：`Archivo Black`（系统已有，强设计感），放进黄色实色块里。
- **中文**：`PingFang SC` Semibold + `-webkit-text-stroke` 描边加重（系统无 Heavy 中文）。
- **升级**：把 `MiSans Heavy` / `思源黑体 Heavy` 放进 `assets/fonts/` 接 `@font-face` → 中文才够狠。这是中文设计感的最后一块。

## 5. 标题

- 砍狠：主标题 8–16 字，最多 3 行，每行短。
- 关键词 1–2 个，用 `[方括号]` → 黄色实色块。例：`我把 [Codex]\n做成封面 [Skill]`。
- 强可读：白字描边 + 投影，压在原图上也清楚。
- 副标题可省；有就小一号。

## 6. 人物

- 喂**真人照片**（带环境更好）。3D 渲染 / logo / 插画抠不出来。
- 抠图整幅、边缘干净；顶层人物加细描边脱离背景。
- 人物站一侧（`person_side`），另一侧留给标题。
- 横版(16:9/4:3)从竖图取景会裁上下，用 `focus` 的 object-position 保住头部（默认 `50% 30%`）。

## 7. 品牌签名

- 左下角**只放「@鱼亦乐」文字**，小号，**不放 logo 图标**。
- 固定位置、固定字 = 记忆点。

## 8. 多比例（重排，不是裁切）

- **16:9**（B站/YouTube）：标题左、人物右。横图原生比例，效果最佳。
- **4:3**：同 16:9 思路，人物更居中，标题要缩一点别被挡。
- **3:4**（抖音/视频号竖）：竖图来源 → 全铺、标题压上方、人物下方居中；横图来源 → `render.py` 自动切「上方标题带 + 下方照片带」，避免裁竖图放大撞脸。
- 各比例字号 / focus 独立调，不是同一张图裁切。
- 拍摄建议：**16:9 用横图、3:4 优先用竖图**最稳；但横竖来源现在都能出三比例。

## 8.5 两套模板与各自规则

两套共享 pop-out 三层 + @鱼亦乐 签名（同一频道），但视觉语言不同，**不在一张图里混用**（`brief.template` 切换）。

**`popout`（海报风，默认）**：大白字 + 黄色实色块关键词。强冲击，覆盖大多数视频。

**`ui`（命令面板风）**：标题做成"在 AI 工具里输入的查询" `✦ 查询 ⌘↵`，下面跟结果行。专属规则：
- 查询=hero，**整体放大**；kicker / ⌘↵ / 结果行小字**不放大**。
- 关键词用**蓝**（`#3B82F6`），不用青；少发光、克制。
- 查询要短（命令式词组），别长到溢出撞人物。
- 结果行（"字幕"）实色蓝条**拉满面板宽、右端压到人物后面**形成纵深（横版生效；3:4 上字下图带不压）。

**`paper`（胶片编辑风，浅色）**：褪色(近黑白)照片 + 胶片颗粒/半调网点，深墨标题。专属规则：
- 浅底深字（区别于另两套深底）；照片**褪色**（`saturate(.45)`，保留少量颜色、非近黑白）、`multiply` 融入纸面。质感以**细胶片颗粒(film grain)为主**，半调网点(halftone)只做极细底纹（点要小，别出现明显大圆点）——**中性、不发黄**（别用牛皮纸暖调）。人物褪色**不跳出**、只做氛围。
- 关键词 = **iOS 文本选中**：黄底高亮 `#FFD23F` + 两端蓝色选中手柄 `#2563EB`（`[方括号]` 自动渲染）。
- **iOS 右键菜单浮层**（`menu` 字段）紧贴选中关键词上方、带向下箭头；自动向画面内侧展开避开人物。
- 关键词放在**人物对侧**（人物在右→关键词在左），菜单才有空间。
- 适合观点 / 故事 / 高级感选题，文字是 hero，人物退为背景。

## 9. 验收清单

**合格**（全中才交付）：
- [ ] 眯眼看，只剩人 + 大字 + 强调块（popout 黄 / ui 蓝），都立得住
- [ ] pop-out 三层在：环境可见但退后、人物全亮跳出、文字夹在中间
- [ ] 关键词被人物挡了也仍能读全
- [ ] 背景压暗适度，没压死、也没亮到抢人物
- [ ] 左下「鱼亦乐」纯文字、小、无 logo
- [ ] 三比例各自成立，不是裁切

**不合格**（出现任一就重做）：
- 背景压太死、环境没了 / 压太亮、抢人物
- 关键词被挡到读不出
- 人物像贴纸浮着、没融进原图
- 绿色大字 / 主色超 3 个 / 花
- 出现 logo 图标 / 品牌字太大
- 三比例只是同一张图裁切
