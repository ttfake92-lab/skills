# 输出模板:三件套交付 + 存档约定

报告**给用户看一份**(对话里),**同时存一份**到 benchmarks/。两份内容一致。

---

## 存档约定

路径:`benchmarks/<平台>-<账号名>/<日期>-<标题slug>.md`
- 平台:`douyin` / `xhs` / `bilibili` / `youtube` / `weixin` 等。
- 账号名:用对方昵称,去掉特殊字符。
- 日期:`YYYY-MM-DD`(今天)。
- slug:标题取前 20 字,空格/符号转 `-`。
- 目录不存在就创建。账号级拆解存到 `benchmarks/<平台>-<账号名>/_account-<日期>.md`。

---

## 报告模板(单条内容)

```markdown
# 拆解:<标题>

- 平台 / 粒度:<平台> · 单条
- 链接:<url>
- 作者:<昵称>
- 数据:赞 <> / 藏 <> / 评 <> / 发布 <时间>   (未获取的标注「未获取」)
- 取数方式:WebFetch / 浏览器 / 用户粘贴
- 拆解日期:<日期>

## 一、可复用爆款公式
- **选题公式**:<人群 + 情绪/痛点 + 反常识/利益>
- **结构公式**:<可填空骨架,如 钩子→共情→干货123→反转→CTA>
- **标题/封面公式**:<可复用句式 + 视觉套路>
- **视觉钩子公式**:<景别 + 反差元素 + 屏幕大字 ──── 决定 0.5 秒留人>
- **剪辑节奏公式**:<钩子段 X 秒一切 / 主体段 Y 秒一切 / 结尾 Z 秒一切>
- 👉 套到我:<一个完整改编示例,含选题 + 结构 + 画面想法>

## 二、画面 + 口播逐段拆解(时间轴对齐)

视频整体:时长 <s> / 切点数 <> / 平均 <X> 秒一切 / 主色调 <> / 字幕样式 <>

| 段落 | 时间 | 画面(景别 + 切点 + 字幕) | 口播 / 文案 | 协同方式 | 为什么有效 |
|---|---|---|---|---|---|
| 钩子 | 0–3s | 特写 / 切 2 次 / 红字"<>" | "<原话>" | 预告型(画面剧透下段内容) | 0.5 秒抓眼球 + 5 秒留扣子 |
| 铺垫 | 3–Xs | … | … | 同步/互补/错位/预告/强调 选一 | … |
| 主体 | … | … | … | … | … |
| 高潮/反转 | … | … | … | … | … |
| 收尾/CTA | … | … | … | … | 引导关注/互动 |

> 图文笔记/公众号文章用简化版:「段落 | 原文 + 配图描述 | 在干什么 | 为什么有效」四列。

## 三、人设与内容定位
- 人设标签:<>
- 记忆点 — **听觉**:口头禅 / 价值主张
- 记忆点 — **视觉**:固定开场动作 / 标志服装道具 / 标志场景 / 标志字幕样式
- 内容定位:<赛道 + 差异化打法>

## 最值得我抄的 3 个点
1. …
2. …
3. …
> 下一步建议:<一句可执行,如「下条视频试用这个钩子公式拍 XX 选题」>
```


## 质量自检(交付前过一遍)

- [ ] 三件套齐全:公式、画面+口播逐段拆解、人设定位。
- [ ] 短视频:口播已转写 **且** contact sheet 已读图,缺一不算完整拆解。
- [ ] 公式里**视觉钩子/剪辑节奏公式**已填,不是只有选题/结构。
- [ ] 逐段拆解表里**画面列**有具体描述(景别/切点/字幕),不是写"画面"两个字凑数。
- [ ] 每个公式都有改编示例,含画面想法,不是空泛模板。
- [ ] 数据有出处;未获取的如实标注,没有编造。
- [ ] 结论说清「为什么有效」,不是复述内容/拍了什么。
- [ ] 已存档到 benchmarks/,文件名含平台+账号+日期。
- [ ] 末尾有「最值得抄的 3 个点」+ 可执行下一步。
