# 省份高考模式与选科对照（参考）

本表用于驱动 HTML 第三步（招生信息）的下拉联动。该表是**前端硬编码默认值的依据**——前端代码里也有一份同样的映射；二者出现冲突时，**以省级教育考试院当年正式公告为准**，本表只作历史参考。

新高考全国推进至 2025 年已基本完成。下表只覆盖**新高考省份**——老高考逻辑暂不实现，遇到老高考考生回到 Claude Code 单独处理。

## 3+3 模式（共 6 省/直辖市）

选科规则：从规定的科目集合中**任选 3 科**。

> **跨模式通用约束**：物理与历史不能同时选。3+1+2 是显式首选二选一；3+3 是隐含约束——招生时按"物理类/历史类"分大类下达计划，同时选两个就两类都不能正常报考。前端必须拦截这种组合。

| 省/直辖市 | 启用年份 | 选科池 |
|---|---|---|
| 上海 | 2017 | 物理、化学、生物、政治、历史、地理（6 选 3） |
| 浙江 | 2017 | 物理、化学、生物、政治、历史、地理、**技术**（**7 选 3**） |
| 北京 | 2020 | 物理、化学、生物、政治、历史、地理（6 选 3） |
| 天津 | 2020 | 物理、化学、生物、政治、历史、地理（6 选 3） |
| 山东 | 2020 | 物理、化学、生物、政治、历史、地理（6 选 3） |
| 海南 | 2020 | 物理、化学、生物、政治、历史、地理（6 选 3） |

> **特别注意**：浙江独有「技术」科。前端在用户选浙江时必须显示 7 个选项；其他 3+3 省份只显示 6 个。

## Tier 2 数据可获得性提示

各省考试院公布投档线 / 一分一段表的格式不一，公开互联网上没有任何一个聚合源能稳定提供全部 31 省/区/市的完整可比历史数据。具体策略见 `SKILL.md` 阶段 3 的「Tier 2 现实约束」。

录取建议任务（`application_recommendation`）在用户没有上传本地原始数据时只输出**冲 / 稳 / 保**定性等级，不输出百分数。
