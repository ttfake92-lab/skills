# 录取机会估算协议

此协议用于回答“选择某个专业/专业组，被录取的概率有多少”。目标是可复算和诚实表达不确定性，不是制造一个看似科学的单点数字。

## 1. 可比性闸门

每个历史样本必须与当前申请在以下口径一致：

- 招生省份
- 科类、考试模式或选科要求
- 批次
- 实际投档单位：院校、院校专业组或专业
- 普通类/专项/中外合作等招生类型

专业组重组、批次合并或规则改变时，不直接混算。标记结构变化并缩短可比年份，或降级为定性判断。

## 2. 数据字段

对每年记录：

- 年份
- 投档最低分
- 投档最低位次
- 本科类考生人数或可比人数（若官方公布）
- 招生计划数
- 专业组与专业构成
- 来源 ID

当前用户记录成绩、位次、当年考生人数和当年计划数。

## 3. 标准化

优先比较位次。不同年份考生规模变化明显且有官方人数时，同时计算：

`位次百分位 = 位次 / 可比考生人数`

对每个历史年份判断：用户当前位次百分位是否优于该年投档最低位次百分位。没有官方可比人数时使用原始位次，并明确限制。

## 4. 基础经验结果

展示：

- `历史跨线次数 / 可比年份数`
- 每年的位次安全边际
- 最近年份趋势
- 招生计划增减

历史跨线频率是证据摘要，不直接等同于真实录取概率。

## 5. 数字区间

满足至少 3 个可比年份、当年计划已发布、无重大规则变化时，才可输出模型估算区间。

默认使用保守的 Beta-Binomial 区间表达小样本不确定性：

- 设可比年份数为 `n`
- 用户按当前标准化位次可跨线的年份数为 `k`
- 使用均匀先验 `Beta(1,1)`
- 后验为 `Beta(k+1, n-k+1)`
- 输出后验中位数附近的整数百分比区间，并至少展示 80% 可信区间

这只是“在历史年份波动可代表未来”的基础模型。页面必须公开 `n`、`k`、区间和假设。

计划数、专业组变化或趋势不得偷偷塞入主观权重。分别做情景分析：

- 乐观情景
- 基准情景
- 保守情景

若需要更复杂模型，必须保留可执行代码、输入数据和测试结果，并让独立审计 agent 复算。

## 6. 输出格式

同时展示：

- 模型估算区间
- 风险层：冲刺 / 有机会 / 较稳 / 兜底
- `k/n` 历史跨线情况
- 可比年份
- 最大不确定性
- 每个输入的官方来源

不要输出小数点概率。不要使用“录取概率 100%”“稳录”等保证性措辞。

## 7. 强制降级

出现任一情况时不输出数字概率：

- 少于 3 个可比年份
- 当年招生计划未发布
- 找不到官方最低位次
- 专业组或招生规则发生重大变化且无法映射
- 来源冲突未解决
- 独立审计无法复算

此时只输出定性等级、现有证据和待补数据。
