# SOURCES.md 格式

```md
# College Application 来源台账

最后核验：YYYY-MM-DD

| ID | 声明/数据 | 发布机构 | 完整 URL | 发布/数据年份 | 适用范围 | 来源级别 | 最后核验 | 状态 |
|---|---|---|---|---|---|---|---|---|
| SRC-001 | | | https://... | | | 官方原始 / 官方二手 / 学术来源 / 辅助 | | 已核验 / 待核验 / 无法核验 / 失效 / 冲突 |

## 冲突与口径
- 

## 无法查证或待核验
- [ ] 
```

规则：

- 每个外部事实、理论依据、行业结论、专业信息、投档数据和概率输入都引用一个或多个来源 ID。
- 性格测试理论来源（RIASEC、Mini-IPIP、O*NET 等）也必须进入来源台账。
- URL 必须指向实际页面、论文 DOI 或文件，不使用搜索结果页替代。
- 记录省份、年份、科类、批次、专业组、量表版本、专业目录版本等适用口径。
- 页面失效时保留历史记录并寻找官方替代来源，不静默换链。
- 辅助来源不能单独支撑招生规则、专业代码、投档线、退档条件或录取机会判断。
- 无法访问、无法确认发布机构、年份/口径不匹配或来源互相冲突时，状态必须标为“无法核验”或“冲突”，并在最终报告中明确告知用户。
